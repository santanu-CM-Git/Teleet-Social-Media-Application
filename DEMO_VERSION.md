# Demo Version Notice

This is a demo version of the Teleet application. The code structure is visible for demonstration and review purposes.

## Important Notes

1. **This code cannot be run** without proper backend API configuration, Firebase setup, and third-party service credentials.

2. **Sensitive credentials have been removed** and replaced with placeholders:
   - API keys
   - Firebase configuration
   - Agora credentials
   - RevenueCat keys
   - Branch.io keys
   - AdMob identifiers

3. **Backend API is required** - The application requires a backend server running at the configured base URL.

4. **Third-party services** must be configured:
   - Firebase project setup
   - Agora account and credentials
   - RevenueCat account
   - Google Mobile Ads account
   - SightEngine API account

## Purpose

This demo version allows you to:
- Review the code structure and architecture
- Understand the application features and functionality
- See how different components are organized
- Review API endpoints and data models

## What's Included

- Complete Flutter project structure
- All screen implementations (UI code)
- API service layer structure
- Data models
- State management implementation
- Navigation structure
- Localization files

## What's Not Included / Removed

- Sensitive API keys and secrets
- Firebase configuration files (sanitized)
- Backend server code
- Database schemas
- Server-side API implementation

## Getting Started (For Review Only)

This code is provided for review purposes only. To actually run this application, you would need:

1. Set up Firebase project and download configuration files
2. Configure all API keys in `lib/utilities/const.dart`
3. Set up backend API server
4. Configure all third-party services
5. Set up proper build configurations for Android and iOS
